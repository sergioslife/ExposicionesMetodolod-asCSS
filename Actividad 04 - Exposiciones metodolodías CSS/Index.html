¿Qué es BEM (Block, Element, Modifier)?

La metodología BEM, que significa Block, Element, Modifier (Bloque, Elemento, Modificador), fue desarrollada por Yandex como una forma de mejorar la mantenibilidad y escalabilidad del código CSS. BEM se enfoca en estructurar y nombrar clases CSS de manera que el código sea más modular y fácil de entender.

  • Bloques (Block): Un componente independiente que tiene un significado completo por sí mismo. Puede ser cualquier parte de la interfaz de usuario, como un botón, un menú, o un formulario. El bloque es autónomo y no debe depender de otros componentes.
    • .menu, .boton, .caja-de-busqueda
  • Elementos (Element): Parte de un bloque que cumple una función específica dentro de ese bloque. Los elementos no existen fuera del bloque y están relacionados únicamente con él.
    • .menu__item, .boton__icono, .caja-de-busqueda__input
  • Modificadores (Modifier): Variaciones o estados de un bloque o un elemento que cambian su apariencia o comportamiento sin alterar su estructura básica.
    • .menu--horizontal, .boton--grande, .boton--activo

¿Qué es OOCSS (Object-Oriented CSS)?

La metodología OOCSS (CSS Orientado a Objetos) fue introducida por Nicole Sullivan. Está basada en los principios de la programación orientada a objetos, donde se busca separar la estructura visual (estilos de contenedores) de la estructura del contenido (estilos de objetos). El objetivo de OOCSS es crear CSS reutilizable y mantenible.

  • Separar estructura y estilo: OOCSS promueve la creación de clases que manejen el aspecto visual de un elemento (como el tamaño o el color) de forma independiente de su estructura HTML. Por ejemplo, en lugar de aplicar el color y la disposición directamente a los elementos, creamos clases separadas para cada uno.
  • Crear objetos reutilizables: Separa los estilos de los contenedores (envoltorios o "wrappers") y los estilos del contenido, para que un mismo objeto pueda ser reutilizado en diferentes contextos sin duplicar código.
